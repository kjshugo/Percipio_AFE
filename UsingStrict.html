JavaScript STRICT Mode (JavaScript variables must be defined!)

Strct Mode also stops using pre-defined special (or 'reserved words')as variables (eg. undefined)

Variable Declarations: var, let and const

var : defines variables which can be re-defined (be careful!)

let : defines variables that can NOT be re-defined!

const :defines constant value to a named variable and cannot be updated, changed or re-declared.
Contant variables best practice is to use CAPS for them (eg. MY_BIRTHDAY = "15/08/1968").

Variable Hoisting is using a variable before it is defined. (Fuck knows why you'd do this!).
This can only be done with the 'var' command ('let' and 'const' will throw an error)

Empty Values in JavaScript: undefined and null / Conversions

Variables (var and let) that are defined but with no set value are implicintly given the value 'undefined'.
The 'null' value has to be explicitly defined when creating variables.

JavaScript implicitly converts numbers to strings if used in calculations (left to right)
eg. var x = "5" + 2 + 3 will equal 523 (string)
and var y = 2 + 3 + "7" will equal 57 (string - but does the number calc first)
'let' works the same way.

User Interactions Using alert, prompt and confirm functions

alert : places an alert on the screen that the user can only acknowledge.
prompt : puts an input box on the screen that the user can interact with and returns a value that's entered.
confirm : puts an option box a user can confirm 'OK' (returns true) or cancel (returns false)
